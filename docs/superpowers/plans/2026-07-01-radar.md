# Radar Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development. Steps use checkbox (`- [ ]`) syntax.

**Goal:** Build Radar — a student-attendance app: teachers mark presence on mobile, admins see attendance/absenteeism charts on mobile+desktop. **Front-only experiment: all data lives in `localStorage`** (no backend for now).

**Architecture:** Next.js 16 App Router SPA (static export), Feature-Sliced Design. A typed `localStorage` store is the data layer, seeded with demo data. Entity fetchers are async with stable signatures so a Supabase backend can replace the store later without touching features. Analytics computed in JS over the store.

**Tech Stack:** Next.js 16, React 19, TypeScript, Tailwind CSS 4, pnpm, Radix/shadcn, recharts, TanStack Query, zustand, zod, react-hook-form, Playwright, Vitest.

## Global Constraints

- UI language: Brazilian Portuguese in 100% of user-visible text.
- Theme: light mode only. `brand-500` = `#2563eb`, `accent` = `#f59e0b`.
- Teacher UX is **mobile-first**; admin UX is **responsive mobile + desktop**.
- **Data layer = `localStorage`** (key `radar.db.v1`), seeded with demo data on first load. No network calls. Entity fetchers return Promises with signatures stable enough to swap in Supabase later.
- FSD import rule: only import from layers below (`app → widgets → features → entities → shared`). Never upward/lateral.
- TypeScript: explicit public types; no `any` (`unknown` + narrowing); props are named interfaces. Semantic names, no one-letter identifiers.
- Git: atomic micro-commits, English imperative messages; **never** mention Claude/Anthropic/AI/agent, never `Co-Authored-By`; author is **Emerson Silva <emerson_jdss@hotmail.com>** (commit with `git -c user.name="Emerson Silva" -c user.email="emerson_jdss@hotmail.com"`). Never `git push`.
- Tests: unit tests for pure logic (store, analytics, models). E2E with Playwright against the built app + seeded localStorage, with PNG evidence in `e2e/<feature>/evidencias/`. (No MSW — there is no network to mock.)

```ts
export type StatusPresenca = "presente" | "ausente" | "atrasado" | "justificado";

  nome: string;
  email: string;
  papel: Papel;

  id: string;
  nome: string;
  serie: string;

  nome: string;
  matricula: string;
  turmaId: string;
  ativo: boolean;

  id: string;

  chamadaId: string;

}

  presencas: Presenca[];
}

export function loadDb(): RadarDb; // seeds if empty
export function saveDb(db: RadarDb): void;
export function resetDb(): void; // wipe + reseed (for demo/E2E)

// analytics.ts — pure functions over a RadarDb snapshot
export function frequenciaPorTurma(
  db: RadarDb,
): { turmaId: string; turmaNome: string; total: number; presentes: number; pct: number }[];
export function frequenciaPorAluno(
  db: RadarDb,
  turmaId: string,
): {
  alunoId: string;
  alunoNome: string;
  matricula: string;
  total: number;
  faltas: number;
  pct: number;
}[];
export function tendenciaAbsenteismo(
  db: RadarDb,
  desde: string,
): { data: string; pctAusencia: number }[];
export function alunosEmRisco(
  db: RadarDb,
  limite: number,
): { alunoId: string; alunoNome: string; turmaNome: string; pctFalta: number }[];
```
