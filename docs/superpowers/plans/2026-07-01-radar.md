# Radar Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Build Radar — a student attendance app where teachers mark presence on mobile and admins see attendance/absenteeism charts on mobile+desktop.

**Architecture:** Next.js 16 App Router SPA (static export) using Feature-Sliced Design, talking directly to Supabase (Postgres + RLS + RPCs). No custom server. Analytics are SQL views/RPCs; attendance is the source of truth on the server.

**Tech Stack:** Next.js 16, React 19, TypeScript, Tailwind CSS 4, pnpm, Radix/shadcn, recharts, TanStack Query, zustand, zod, react-hook-form, `@supabase/ssr`, Playwright, Vitest, MSW.

## Global Constraints

- UI language: Brazilian Portuguese in 100% of user-visible text.
- Theme: light mode only.
- `brand-500` = `#2563eb` (blue — "radar" signal), `accent` = `#f59e0b`.
- Teacher UX is **mobile-first**; admin UX is **responsive mobile + desktop**.
- FSD import rule: only import from layers below (`app → widgets → features → entities → shared`). Never upward, never lateral between slices of the same layer.
- TypeScript: explicit types on public component interfaces; no `any` (use `unknown` + narrowing); props always a named interface.
- No `service_role` key in the frontend — only the publishable key; real protection is Postgres RLS.
- Git: atomic micro-commits, English imperative messages (`add X`, `fix Y`); **never** mention Claude/Anthropic/AI/agent, never `Co-Authored-By`, human is the author. Never `git push` without explicit human confirmation.
- Every SDD feature must have unit + MSW-integration + Playwright E2E (with PNG evidence in `e2e/<feature>/evidencias/`).

## Task 5: RLS policies

**Files:**
- Create: `supabase/migrations/0002_rls.sql`

**Interfaces:**
- Produces: `is_admin()` SQL helper; RLS enforcing teacher-scope and admin-all.

- [ ] **Step 1: Write `0002_rls.sql`:**

```sql
create or replace function is_admin() returns boolean
language sql security definer stable set search_path = public as $$
  select exists (select 1 from perfis where id = auth.uid() and papel = 'admin');
$$;

alter table perfis enable row level security;
alter table turmas enable row level security;
alter table alunos enable row level security;
alter table chamadas enable row level security;
alter table presencas enable row level security;

-- perfis: user sees own profile; admin sees all
create policy perfis_select on perfis for select using (id = auth.uid() or is_admin());
create policy perfis_admin_write on perfis for all using (is_admin()) with check (is_admin());

-- turmas: teacher sees own; admin all
create policy turmas_select on turmas for select using (professor_id = auth.uid() or is_admin());
create policy turmas_admin_write on turmas for all using (is_admin()) with check (is_admin());

-- alunos: visible if turma belongs to teacher or admin
create policy alunos_select on alunos for select using (
  is_admin() or exists (select 1 from turmas t where t.id = alunos.turma_id and t.professor_id = auth.uid())
);
create policy alunos_admin_write on alunos for all using (is_admin()) with check (is_admin());

-- chamadas: teacher manages own turma's chamadas; admin all
create policy chamadas_rw on chamadas for all using (
  is_admin() or exists (select 1 from turmas t where t.id = chamadas.turma_id and t.professor_id = auth.uid())
) with check (
  is_admin() or exists (select 1 from turmas t where t.id = chamadas.turma_id and t.professor_id = auth.uid())
);

-- presencas: via parent chamada's turma ownership; admin all
create policy presencas_rw on presencas for all using (
  is_admin() or exists (
    select 1 from chamadas c join turmas t on t.id = c.turma_id
    where c.id = presencas.chamada_id and t.professor_id = auth.uid())
) with check (
  is_admin() or exists (
    select 1 from chamadas c join turmas t on t.id = c.turma_id
    where c.id = presencas.chamada_id and t.professor_id = auth.uid())
);
```

## Task 6: Analytics RPCs/views

**Files:**
- Create: `supabase/migrations/0003_analytics.sql`
- Modify: `src/shared/lib/supabase/types.ts` (regenerate)

**Interfaces:**
- Produces RPCs (all respect RLS via `security invoker`):
  - `frequencia_por_turma()` → rows `{ turma_id uuid, turma_nome text, total int, presentes int, pct numeric }`
  - `frequencia_por_aluno(p_turma_id uuid)` → `{ aluno_id, aluno_nome, matricula, total int, faltas int, pct numeric }`
  - `tendencia_absenteismo(p_desde date)` → `{ data date, pct_ausencia numeric }`
  - `alunos_em_risco(p_limite numeric)` → `{ aluno_id, aluno_nome, turma_nome, pct_falta numeric }`

- [ ] **Step 1: Write `0003_analytics.sql`** — define the four functions. Faltas = status in ('ausente','justificado'); presença conta 'presente' e 'atrasado'. Example (the pattern for all four):

```sql
create or replace function frequencia_por_turma()
returns table (turma_id uuid, turma_nome text, total bigint, presentes bigint, pct numeric)
language sql security invoker stable set search_path = public as $$
  select t.id, t.nome,
         count(p.*) as total,
         count(*) filter (where p.status in ('presente','atrasado')) as presentes,
         round(100.0 * count(*) filter (where p.status in ('presente','atrasado'))
               / nullif(count(p.*),0), 1) as pct
  from turmas t
  left join chamadas c on c.turma_id = t.id
  left join presencas p on p.chamada_id = c.id
  group by t.id, t.nome
  order by t.nome;
$$;

create or replace function alunos_em_risco(p_limite numeric default 25)
returns table (aluno_id uuid, aluno_nome text, turma_nome text, pct_falta numeric)
language sql security invoker stable set search_path = public as $$
  select a.id, a.nome, t.nome,
         round(100.0 * count(*) filter (where p.status in ('ausente','justificado'))
               / nullif(count(p.*),0), 1) as pct_falta
  from alunos a
  join turmas t on t.id = a.turma_id
  left join chamadas c on c.turma_id = a.turma_id
  left join presencas p on p.chamada_id = c.id and p.aluno_id = a.id
  group by a.id, a.nome, t.nome
  having round(100.0 * count(*) filter (where p.status in ('ausente','justificado'))
               / nullif(count(p.*),0), 1) >= p_limite
  order by pct_falta desc;
$$;
```
